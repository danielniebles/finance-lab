# Finance Lab — Design Context

## Design Context

### Users
Single user (the developer/owner). Uses the app in two modes: quick daily glances to check status, and focused monthly sessions to import data, review analysis, and plan. Desktop-only, dark environment, unhurried but purposeful.

### Brand Personality
**Structured, serious, precise.** Engineering-minded — everything has a place, data is sacred, UI serves the numbers. Not a consumer fintech product. Not trying to delight strangers. Built for someone who respects the craft of knowing where their money goes.

### Aesthetic Direction
Dark-first (already in place). Neutral palette with teal/green primary (already OKLCH-based — keep it). No flashy fintech aesthetics, no gradient hero metrics, no decorative chrome. The interface should feel like a well-maintained instrument panel — everything visible has a reason to be there.

Anti-references: Mint, YNAB (too consumer/colorful), generic SaaS dashboards with gradient cards and hero metrics.

### Design Principles
1. **Data first** — typography, spacing, and layout exist to make numbers readable, not to decorate them
2. **Density with clarity** — information-dense layouts are acceptable, but never at the cost of scanability
3. **Earned color** — color signals meaning (green=good, red=bad, amber=warning); decorative color is a distraction
4. **No UI noise** — every element must justify its presence; borders, shadows, and containers only when they add structure
5. **Consistent rhythm** — predictable spacing and component patterns reduce cognitive load during monthly review sessions
